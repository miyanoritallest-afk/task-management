# タスク管理アプリ 要件定義書

作成日：2026-04-24

---

## 1. システム概要・背景

本アプリはWebフロントエンド開発の講座課題として作成するタスク管理ツールである。  
Trello風のカンバンボード形式を採用し、タスクをカード形式で管理することで進捗状況を視覚的に把握できることを目的とする。  
講座内でNext.js・Tailwind CSS・Supabase（PostgreSQL）・Prismaを用いたUIコンポーネント設計・状態管理・データベース設計・データ永続化の実装スキルを習得する。

## 4. 画面仕様の詳細

### 4-1. ボード画面（メイン画面）

| 要素 | 仕様 |
|------|------|
| ヘッダー | アプリ名「タスクボード」を表示。フェーズ4では現在のボード名も表示 |
| カラム | 「Todo」「進行中」「完了」の3列を横並びで表示。各カラムはスクロール可能 |
| カラムヘッダー | カラム名とカード枚数（例：Todo (3)）を表示 |
| カード | タイトルを表示。フェーズ2以降は期限も表示。期限超過時は赤字で表示 |
| カード追加ボタン | 各カラム末尾に「+ カードを追加」ボタンを配置 |

### 4-2. カード追加

| 要素 | 仕様 |
|------|------|
| トリガー | 「+ カードを追加」ボタンを押す |
| 入力UI | カラム末尾にテキスト入力欄とボタンをインラインで表示 |
| 追加ボタン | 入力欄が空でない場合のみ活性化。押下でカードを該当カラムの末尾に追加 |
| キャンセル | Escキーまたは「×」ボタンで入力欄を閉じる |
| 保存タイミング | 追加と同時にSupabaseへ保存 |

### 4-3. カード詳細（フェーズ2）

| 要素 | 仕様 |
|------|------|
| トリガー | カードのタイトル部分をクリック |
| 表示形式 | モーダルダイアログで表示 |
| 編集項目 | タイトル（テキスト入力）・説明文（テキストエリア）・期限日（日付ピッカー） |
| 保存 | 「保存」ボタンで変更を反映しSupabaseを更新 |
| 閉じる | 「×」ボタンまたはモーダル外クリックで閉じる（未保存の場合は変更を破棄） |

### 4-4. カード操作

| 操作 | 仕様 |
|------|------|
| 左に移動 | カード左端の「←」ボタン。Todoカラムでは非表示 |
| 右に移動 | カード右端の「→」ボタン。完了カラムでは非表示 |
| 削除 | カード右上の「×」ボタン。押下で即削除（確認ダイアログなし） |
| ドラッグ&ドロップ（フェーズ3） | カードをつかんで別カラムまたは同カラム内の別位置にドロップして移動 |

### 4-5. 画面遷移図

```
[ボード画面]
    │
    ├── 「+ カードを追加」クリック → [カード追加入力欄（インライン）]
    │       │
    │       ├── 追加ボタン押下 → カード追加 → [ボード画面]
    │       └── Esc / × → [ボード画面]
    │
    └── カードタイトルクリック（フェーズ2） → [カード詳細モーダル]
            │
            ├── 保存ボタン押下 → [ボード画面]
            └── × / モーダル外クリック → [ボード画面]
```

### 5-2. ユースケース詳細

#### UC-01: カードを追加する

| 項目 | 内容 |
|------|------|
| **概要** | ユーザーがカラムにカードを新規追加する |
| **事前条件** | ボード画面が表示されている |
| **基本フロー** | 1. 任意のカラムの「+ カードを追加」ボタンを押す<br>2. テキスト入力欄が表示される<br>3. タイトルを入力する<br>4. 追加ボタンを押す<br>5. カードがカラム末尾に追加され、DBに保存される |
| **代替フロー** | 3a. Escキーまたは「×」ボタンを押す → 入力欄が閉じ、カードは追加されない |
| **事後条件** | 入力したタイトルのカードがカラムに表示される |

#### UC-02: カードを削除する

| 項目 | 内容 |
|------|------|
| **概要** | ユーザーがカードを削除する |
| **事前条件** | 削除対象のカードが存在する |
| **基本フロー** | 1. 削除したいカードの「×」ボタンを押す<br>2. カードが即座に削除され、DBから削除される |
| **代替フロー** | なし（確認ダイアログなし） |
| **事後条件** | 対象カードがカラムから消える |

#### UC-03: カードを別のカラムに移動する

| 項目 | 内容 |
|------|------|
| **概要** | ユーザーがボタン操作でカードを隣のカラムに移動する |
| **事前条件** | 移動対象のカードが存在する |
| **基本フロー** | 1. カードの「←」または「→」ボタンを押す<br>2. カードが隣のカラムの末尾に移動する<br>3. DBのcolumn_idが更新される |
| **代替フロー** | 1a. Todoカラムのカードは「←」ボタンが非表示<br>1b. 完了カラムのカードは「→」ボタンが非表示 |
| **事後条件** | カードが移動先のカラムに表示される |

#### UC-04: カードの詳細を編集する（フェーズ2）

| 項目 | 内容 |
|------|------|
| **概要** | ユーザーがカードのタイトル・説明文・期限を編集する |
| **事前条件** | 編集対象のカードが存在する |
| **基本フロー** | 1. カードのタイトル部分をクリックする<br>2. カード詳細モーダルが表示される<br>3. タイトル・説明文・期限日を編集する<br>4. 「保存」ボタンを押す<br>5. 変更内容がDBに保存され、カードの表示が更新される |
| **代替フロー** | 4a. 「×」ボタンまたはモーダル外をクリック → 変更を破棄してモーダルを閉じる |
| **事後条件** | 編集内容がカードに反映される |

#### UC-05: カードに期限を設定する（フェーズ2）

| 項目 | 内容 |
|------|------|
| **概要** | ユーザーがカードに締切日を設定する |
| **事前条件** | カード詳細モーダルが表示されている |
| **基本フロー** | 1. 日付ピッカーで締切日を選択する<br>2. 「保存」ボタンを押す<br>3. カードに期限日が表示される |
| **代替フロー** | 3a. 期限日が過去の日付の場合、カード上に赤字で表示される |
| **事後条件** | カードに期限日が表示される |

#### UC-06: カードをドラッグ&ドロップで移動する（フェーズ3）

| 項目 | 内容 |
|------|------|
| **概要** | ユーザーがカードをドラッグして別のカラムまたは同カラム内の別位置に移動する |
| **事前条件** | 移動対象のカードが存在する |
| **基本フロー** | 1. カードをつかむ（マウスボタンを押したまま）<br>2. 移動先にドラッグする<br>3. ドロップする<br>4. カードが移動先に表示され、DBのcolumn_idとpositionが更新される |
| **代替フロー** | 3a. カードをもとの位置にドロップ → 移動しない |
| **事後条件** | カードが指定した位置に表示される |

#### UC-07: カラム内でカードを並び替える（フェーズ3）

| 項目 | 内容 |
|------|------|
| **概要** | ユーザーが同一カラム内でカードの順序を変更する |
| **事前条件** | 同一カラムに2枚以上のカードが存在する |
| **基本フロー** | 1. カードをドラッグして同カラム内の別位置にドロップする<br>2. カードの順序が変わり、DBのpositionが更新される |
| **代替フロー** | なし |
| **事後条件** | カラム内のカード順序が変更される |

#### UC-08: ボードを新規作成する（フェーズ4）

| 項目 | 内容 |
|------|------|
| **概要** | ユーザーが新しいボードを作成する |
| **事前条件** | ボード画面が表示されている |
| **基本フロー** | 1. 「新しいボードを作成」ボタンを押す<br>2. ボード名を入力する<br>3. 作成ボタンを押す<br>4. 新しいボードが作成され、DBに保存される<br>5. 作成したボードに切り替わる |
| **代替フロー** | 2a. ボード名が空の場合、作成ボタンを非活性にする |
| **事後条件** | 新しいボードが表示される |

#### UC-09: ボードを切り替える（フェーズ4）

| 項目 | 内容 |
|------|------|
| **概要** | ユーザーが既存のボードを選択して切り替える |
| **事前条件** | 2つ以上のボードが存在する |
| **基本フロー** | 1. ヘッダーのボード選択メニューを開く<br>2. 切り替えたいボードを選択する<br>3. 選択したボードのカラムとカードが表示される |
| **代替フロー** | なし |
| **事後条件** | 選択したボードの内容が表示される |

#### UC-10: カードに色をつける（フェーズ4）

| 項目 | 内容 |
|------|------|
| **概要** | ユーザーがカードに色を設定してラベル付けする |
| **事前条件** | カード詳細モーダルが表示されている |
| **基本フロー** | 1. カラーピッカーで色を選択する<br>2. 「保存」ボタンを押す<br>3. カードが選択した色で表示される |
| **代替フロー** | 1a. 色をクリアする → カードがデフォルト色に戻る |
| **事後条件** | カードが指定した色で表示される |

## 6. データ構造・ER図

### 5-1. エンティティ概要

| テーブル名 | 説明 |
|-----------|------|
| boards | ボード（フェーズ4で複数ボードに対応） |
| columns | カラム（Todo / 進行中 / 完了） |
| cards | カード（タスク本体） |

### 5-2. ER図

```
boards
─────────────────────────────
PK  id          UUID
    name        VARCHAR(100)
    created_at  TIMESTAMP
    updated_at  TIMESTAMP

        │
        │ 1 : N
        ▼

columns
─────────────────────────────
PK  id          UUID
FK  board_id    UUID  ──→ boards.id
    name        VARCHAR(50)
    position    INTEGER
    created_at  TIMESTAMP
    updated_at  TIMESTAMP

        │
        │ 1 : N
        ▼

cards
─────────────────────────────
PK  id          UUID
FK  column_id   UUID  ──→ columns.id
    title       VARCHAR(100)
    description TEXT
    due_date    DATE
    color       VARCHAR(20)
    position    INTEGER
    created_at  TIMESTAMP
    updated_at  TIMESTAMP
```

### 5-3. リレーション説明

| リレーション | 種別 | 説明 |
|-------------|------|------|
| boards → columns | 1対多 | 1つのボードは複数のカラムを持つ |
| columns → cards | 1対多 | 1つのカラムは複数のカードを持つ |

### 5-4. 各カラムの説明

**boards テーブル**

| カラム名 | 型 | 説明 |
|---------|-----|------|
| id | UUID | 主キー。自動生成 |
| name | VARCHAR(100) | ボード名 |
| created_at | TIMESTAMP | 作成日時 |
| updated_at | TIMESTAMP | 更新日時 |

**columns テーブル**

| カラム名 | 型 | 説明 |
|---------|-----|------|
| id | UUID | 主キー。自動生成 |
| board_id | UUID | 所属するボードのID（外部キー） |
| name | VARCHAR(50) | カラム名（Todo / 進行中 / 完了） |
| position | INTEGER | カラムの表示順（左から0, 1, 2） |
| created_at | TIMESTAMP | 作成日時 |
| updated_at | TIMESTAMP | 更新日時 |

**cards テーブル**

| カラム名 | 型 | 説明 |
|---------|-----|------|
| id | UUID | 主キー。自動生成 |
| column_id | UUID | 所属するカラムのID（外部キー） |
| title | VARCHAR(100) | カードのタイトル |
| description | TEXT | カードの説明文（フェーズ2） |
| due_date | DATE | 締切日（フェーズ2） |
| color | VARCHAR(20) | カードの色（フェーズ4） |
| position | INTEGER | カラム内での表示順 |
| created_at | TIMESTAMP | 作成日時 |
| updated_at | TIMESTAMP | 更新日時 |

## 6. 非機能要件

### 6-1. パフォーマンス

| 項目 | 要件 |
|------|------|
| 初期表示速度 | ページロードから表示完了まで2秒以内 |
| 操作レスポンス | カード追加・移動・削除の反映は即時（楽観的更新を使用） |
| カード上限 | 1カラムあたり最大100枚を想定（それ以上は動作保証外） |

### 6-2. 対応ブラウザ

| ブラウザ | バージョン |
|---------|-----------|
| Google Chrome | 最新版 |
| Microsoft Edge | 最新版 |
| Firefox | 最新版 |
| Safari | 最新版（macOS） |

モバイルブラウザは動作確認対象外とする。

### 6-3. 対応デバイス・レイアウト

| 項目 | 要件 |
|------|------|
| 対象デバイス | PCブラウザのみ（スマートフォン・タブレット非対応） |
| 最小画面幅 | 1024px以上を推奨 |
| レイアウト | 3カラム横並び固定。1024px未満では横スクロールで対応 |

### 6-4. データ永続化・セキュリティ

| 項目 | 要件 |
|------|------|
| 保存場所 | Supabase（PostgreSQL）。Next.jsのAPI Route経由でアクセス |
| ORM | Prismaを使用してデータアクセスを型安全に管理 |
| セキュリティ | Supabaseの接続キーは環境変数で管理し、クライアントに露出させない |
| XSS対策 | ユーザー入力はテキストとして扱い、HTMLとして評価しない |
| デプロイ | Vercelにホスティング。環境変数はVercelの管理画面で設定 |

### 6-5. アクセシビリティ

| 項目 | 要件 |
|------|------|
| キーボード操作 | Escキーでモーダル・入力欄を閉じられること |
| フォーカス管理 | モーダル表示時はフォーカスをモーダル内に制限する |
| カラーコントラスト | テキストと背景のコントラスト比はWCAG AA基準（4.5:1以上）を目標とする |

## 7. エラーハンドリング

### 7-1. カード追加

| エラーケース | 対応 |
|-------------|------|
| タイトルが空のまま追加しようとした | 追加ボタンを非活性（disabled）にして押せない状態にする |
| タイトルが100文字を超えた | 入力欄の下に「100文字以内で入力してください」と赤字で表示し、追加ボタンを非活性にする |

### 7-2. データベース通信

| エラーケース | 対応 |
|-------------|------|
| Supabaseへの書き込みに失敗した | 画面上部に「データの保存に失敗しました。しばらくしてから再試行してください。」とエラートーストを表示する |
| Supabaseからのデータ取得に失敗した | 「データを読み込めませんでした。ページを再読み込みしてください。」とエラーメッセージを表示する |
| ネットワーク接続が切れた | 「オフラインです。インターネット接続を確認してください。」とエラートーストを表示する |

### 7-3. カード詳細（フェーズ2）

| エラーケース | 対応 |
|-------------|------|
| タイトルを空にして保存しようとした | 保存ボタンを非活性にする |
| 期限日に不正な値が入力された | 日付ピッカーコンポーネントの標準バリデーションに委ねる |

### 7-4. 共通

| エラーケース | 対応 |
|-------------|------|
| 予期しないJavaScriptエラー | コンソールにエラーログを出力する（本課題ではユーザー向け表示は任意） |

## 9. 開発フェーズ

### フェーズ1（基本機能）
- カードの追加・削除
- カラム間の移動（ボタン操作）
- Supabaseへのリアルタイム保存

### フェーズ2（カード詳細）
- カードへの説明文追加
- 締切日の設定と表示

### フェーズ3（操作性向上）
- カラム内の並び替え
- ドラッグ&ドロップでの移動

### フェーズ4（拡張機能）
- 複数ボードの作成・切り替え
- カードへの色付け
